### **Deploy to GitHub Pages** (Choose One Method):

#### **Method A: Personal Access Token** (Recommended)
```bash
# 1. Create token at: https://github.com/settings/tokens/new
#    Scopes: repo, workflow
#    Save the token

# 2. Set remote with token
cd /home/jackal/.openclaw/scaleforge-project
git remote set-url origin https://YOUR_TOKEN_HERE@github.com/jackalAgents/scaleforge-studio.git

# 3. Push gh-pages branch
git push -u origin gh-pages

# 4. Enable GitHub Pages
#    Go to: https://github.com/jackalAgents/scaleforge-studio/settings/pages
#    Source: Deploy from branch
#    Branch: gh-pages / (root)
#    Save

# Site will be live at:
# https://jackalagents.github.io/scaleforge-studio/
```

#### **Method B: SSH Key**
```bash
# 1. Add SSH key to GitHub (if not done)
# 2. Change remote
git remote set-url origin git@github.com:jackalAgents/scaleforge-studio.git

# 3. Push
git push -u origin gh-pages
```

#### **Method C: Web Upload**
1. Go to: https://github.com/jackalAgents/scaleforge-studio
2. Create new branch: `gh-pages`
3. Upload files via web interface:
   - index.html
   - styles.css
   - script.js
   - demos/ folder
   - .gitignore
4. Enable Pages from Settings → Pages → Source: gh-pages
